# Windows (윈도우) - Terraform 설치방법

1. 수동 설치: Terraform 공식 다운로드 페이지에서 Windows용 전용 .zip 파일을 받습니다.

2. 압축을 풀면 terraform.exe 파일 하나가 나옵니다.

3. terraform.exe을 특정 폴더(예: C:\terraform)에 넣고, 해당 경로를 **시스템 환경 변수(Path)**에 추가합니다.

4. 터미널(CMD 또는 PowerShell)에서 terraform -version을 입력해 설치를 확인합니다.

# study.md 내용 바탕으로 terraform 적용

1. Terraform 적용 시 고려할 점 (가이드 반영)
가이드에서는 '직접 클릭'하는 수동 작업을 설명하고 있지만, 앞서 언급하신 Terraform을 사용해 이 환경을 구축한다면 다음과 같은 구성이 필요합니다.

VPC 및 서브넷 설계: EKS는 최소 2개 이상의 가용 영역(AZ)에 걸친 서브넷이 필요합니다. 비용 절감을 위해 가이드의 RDS와 백엔드는 Private Subnet에, 배포용 Bastion이나 로드밸런서(ALB)는 Public Subnet에 배치하도록 코드를 짜야 합니다.

EKS Node Group 사양: 프리티어 범위(t3.micro 등)로 EKS 노드를 구성할 경우, 쿠버네티스 기본 시스템이 차지하는 자원이 많아 실제 앱(FastAPI, Nginx)이 실행될 메모리가 부족할 수 있습니다. 학습용이라면 최소 t3.small 이상을 고려하거나, 불필요한 리소스 제한을 빡빡하게 설정해야 합니다.

2. 가이드 내 '보안 및 네트워크' 보완 사항
ALB 인그레스 컨트롤러 (AWS Load Balancer Controller): EKS에서 외부 접속(80/443 포트)을 받으려면 단순 시큐리티 그룹 설정 외에, AWS 로드밸런서 컨트롤러를 EKS 안에 설치해야 합니다. 이 부분이 누락되면 도커를 띄워도 밖에서 접속이 안 될 수 있습니다.

ECR 인증 문제: GitHub Actions 로봇이 ECR에 이미지를 푸시하려면 IAM 권한뿐만 아니라, aws ecr get-login-password 명령어를 통해 도커 로그인을 수행하는 단계가 .yml 파일에 포함되어야 합니다.

3. 스토리텔링(자소서) 포인트 강화
가이드에 적힌 "왜 바닐라 JS를 썼는가", "왜 MSA인가"에 대한 답변은 매우 훌륭합니다. 여기에 '비용 효율성' 관점을 하나 더 추가하면 기업 직무 면접에서 높은 점수를 받을 수 있습니다.

추가 팁: "무조건 비싼 고사양 서버를 쓰기보다, **트래픽에 따라 서버 대수를 유연하게 조절(HPA: Horizontal Pod Autoscaler)**하고, 사용하지 않는 시간에는 리소스를 최소화하는 설계를 통해 학교 운영 예산을 절감하는 아키텍처를 고민했습니다."라는 내용을 덧붙여 보세요.

4. 실무적 조언 (배포 방식)
가이드에서 언급된 **'블루-그린 배포'**를 EKS에서 구현하려면:

단순히 이미지를 덮어씌우는 방식보다는 ArgoCD 같은 도구를 쓰거나, 쿠버네티스의 Service(Type: LoadBalancer) 설정을 변경하여 트래픽을 구버전에서 신버전으로 한 번에 돌리는 로직이 필요합니다. 처음에는 가장 기본적인 **롤링 업데이트(Rolling Update)**부터 성공시킨 후 블루-그린으로 고도화하는 것을 추천합니다.
